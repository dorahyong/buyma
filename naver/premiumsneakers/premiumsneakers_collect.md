- **사무실 PC에서만 실행**. AWS/서버에서는 네이버 DNS 차단되거나 쿠키 재활용 안 됨.
- **WARP 상태**: buyma 크롤링용 Cloudflare WARP이 켜져 있으면 `smartstore.naver.com` DNS resolve 실패 → 네이버 작업 시 **WARP 꺼야** 함.
- `naver_cookies.json`은 `naver/` 바로 아래(scan_store_brands, premiumsneakers_collector 공용). 쿠키 수명 짧음(몇 시간) → 실행 전 `--login` 권장.
- Playwright `headless=False` 필수 — headless는 네이버 봇 감지에 걸림.

## 리스트 수집(Phase 1) — 페이지네이션 함정

### 실패한 접근 (쓰지 말 것)
1. `?page=N&size=40`: **302 리다이렉트 → ?page=1**로 돌아감. Playwright가 자동 추적해서 항상 p1만 받음. (MD `## list 2page` 섹션이 근거)
2. `?cp=N` 직접 URL navigate: **서버가 200 OK 주지만 HTML 본문은 p1 내용**. 시크릿 창에서 직접 URL 넣어도 마찬가지. 쿠키/Referer/헤더 조합으로도 해결 안 됨.
3. Referer에 `?page=N&size=40` 채워도 효과 없음.
4. brand_url에 이미 `?cp=1`이 있는 상태에서 `?cp=N`/`?page=N`을 덧붙여도 기존 `cp=1`이 이김 → 정규식으로 `cp|page|size|st|dt` 제거 후 재구성이 필요했지만, 결국 URL 방식 자체가 무의미함이 밝혀짐.

### 동작하는 방식
**DOM의 페이지 버튼을 Playwright로 클릭**해야 실제 페이지 전환됨. 네이버 스마트스토어 페이지네이션은 순수 JS 클릭 이벤트 기반.

- 숫자 버튼 셀렉터: `a[data-shp-contents-type="pgn"][data-shp-contents-id="{N}"]`
- "다음" 버튼(6페이지 이상일 때): `a[role="button"]:has-text("다음")`
- **DOM 업데이트 대기**: `page.wait_for_function`으로 첫 상품 ID(`[data-shp-contents-type="chnl_prod_no"]`의 `data-shp-contents-id`)가 바뀔 때까지 대기. `networkidle`만으로는 불안정.

### 총 갯수 파싱
- `"(총 163개)"` 텍스트를 `/총\s*([0-9,]+)\s*개/`로 매칭해서 total 추출
- 클래스명(예: `.eJTgO8xT6T`)은 해시라 **변경 가능** → 텍스트 패턴 매칭 권장
- `max_pages = ceil(total / 40)`로 무한 루프 방지

### 종료 조건 (3중)
1. `page_num >= max_pages` (총 갯수 기반)
2. 숫자 페이지 버튼 + "다음" 버튼 모두 없음
3. 이번 페이지 pnos 전부가 이전 페이지 `seen`과 중복 (= 네이버가 p1 재표시하는 이상 케이스)

### `--skip-existing`과의 상호작용
- 기존 DB에 이미 있는 pno도 **`seen`에는 추가**해야 함 (페이지 재표시 감지용)
- "신규 0건"을 break 조건으로 쓰면 안 됨 (skip-existing에서 전부 스킵되는 정상 경우 오판됨)

## 상세 수집(Phase 2)

- 상품 상세 데이터는 **순수 XHR로만** 전송됨 (HTML에 없음)
- 두 API 동시 호출 후 `page.on('response')` **sync 핸들러**로 캡처:
  - `GET /i/v2/channels/{channelNo}/products/{pno}?withWindow=false` — 상품 정보
  - `POST /i/v2/channels/{channelNo}/product-benefits` — 할인/쿠폰가 (판매가 기준)
- URL 매칭은 **정규식** 사용 (`/products/{pno}/review` 같은 오인 방지)
- `DETAIL_DELAY = (1.0, 3.0)` + 재시도 2회. 연속 호출 시 429 가능 → 429/캡챠 감지 시 중단.
- 이미지는 `productImages` 배열에 중복 포함되는 케이스 있음 → dedup 필수.
- 캡챠 감지: `page.title()`에 `보안` 또는 `captcha` 포함 시 즉시 중단 + `--login` 요청.

## raw_json_data 스키마 (converter 호환)

kasina 공용 converter(`kasina/raw_to_converter_kasina.py` + `--source-site premiumsneakers`)가 기대하는 형태로 저장:

```json
{
  "channel_no": "...",
  "brand_name": "...",
  "brand_id": 0,
  "model_name": "...",
  "options": [
    {"color": "", "tag_size": "EU 39-40", "option_code": "...", "status": "in_stock"}
  ],
  "images": ["https://shop-phinf.pstatic.net/..."],
  "category": "패션잡화 > 남성신발 > 부츠"
}
```

- **`color=""`**: 네이버는 대부분 사이즈만 (신발류) → converter가 FREE로 자동 fallback
- **`tag_size`**: 범위 그대로 저장 (`"EU 39-40"`). BUYMA 수용 여부는 실등록에서 확인 필요.
- **`status`**: 네이버는 수량만 주므로 `stockQuantity > 0`을 `"in_stock"`으로 변환 저장
- **measurements/composition 없음**: converter가 graceful 처리, colorsize_comments가 빈약해짐
