# Cloudflare WARP 도입 보고

작성일: 2026-04-13

## 1. 배경 — 발생한 문제

**BUYMA 자동화 시스템의 핵심 기능인 "경쟁자 최저가 조회"가 중단됨.**

- 증상: `www.buyma.com` 접속 시도 모두 **403 Forbidden** 응답
- 범위: 사무실 공유오피스 IP 전체
- 지속: **3일 이상 계속됨**
- 영향:
  - 일일 자동화 파이프라인의 가격 인하/마진 계산 단계 전부 실패
  - 공유오피스 같은 입주사들도 buyma.com 접속 불가 상태
- 원인: 최근 추가한 `fast_price_updater.py`의 공격적 요청 설정(동시 5개 워커, 0.1초 간격)으로 초당 ~50건 요청 발생 → BUYMA 봇 차단 시스템이 사무실 IP를 블랙리스트 등록

## 2. 제약 조건

- **공유오피스이므로 IP 변경 불가** (공유기 재부팅·통신사 교체 불가)
- BUYMA의 자동 해제를 3일간 기다렸으나 풀리지 않음 → 장기/영구 차단으로 판단
- 사무실 업무 자체에도 지장 (브라우저로도 buyma.com 접속 불가)

## 3. 해결 방안 — Cloudflare WARP

### 3.1 Cloudflare WARP이란
Cloudflare(전 세계 인터넷 트래픽의 약 30%를 처리하는 글로벌 인프라 기업)가 제공하는 **무료 네트워크 우회 서비스**. 일반적으로 "VPN"으로 알려진 기술과 유사하게 동작하지만, 보안·속도·안정성 면에서 업계 최고 수준의 Cloudflare 네트워크를 통해 연결됨.

### 3.2 작동 방식
- PC에서 앱을 켜면 인터넷 트래픽이 Cloudflare 서버를 경유하여 나감
- 외부 사이트(buyma.com 포함)가 보는 IP = 사무실 IP가 아닌 **Cloudflare 서버 IP**
- 차단된 사무실 IP를 우회하여 buyma.com 접근 가능

### 3.3 선택 이유

| 항목 | 상세 |
|---|---|
| **비용** | 무료 (별도 결제·계정 생성 불필요) |
| **속도** | 일반 VPN과 달리 속도 저하 거의 없음 (Cloudflare 글로벌 백본망 사용) |
| **안정성** | 대기업 운영, 24/7 가동 |
| **보안** | 엔터프라이즈급 암호화, Cloudflare의 기업 보안 기준 준수 |
| **도입 난이도** | 설치·설정 5분 이내 |

## 4. 적용 결과

- WARP 설치 후 즉시 **buyma.com 정상 접속 확인** (브라우저·자동화 스크립트 모두)
- 일일 자동화 파이프라인 재개 가능 상태로 복구

## 5. 재발 방지 조치 (병행)

단순히 WARP로 우회만 하면 같은 방식으로 WARP IP까지 차단될 수 있으므로, **근본 원인이었던 공격적 요청 설정도 함께 개선**함.

| 설정값 | 이전 (차단 유발) | 개선 후 |
|---|---|---|
| 동시 워커 수 | 5개 | **3개** |
| 요청 간격 | 0.1초 고정 | **0.1~0.3초 랜덤** |
| 초당 요청 수 | 약 50건 | **약 15건** (과거 안전 운영 수준) |

## 6. 장기 리스크 및 대책

### 단기 (현재)
- WARP로 우회 운영 중, 정상 작동
- 설정 보수화로 차단 재발 가능성 낮춤

### 중장기 잠재 리스크
1. **WARP IP 버닝**: 크롤링 봇들이 Cloudflare IP를 공격적으로 쓰면 BUYMA가 WARP 대역 전체를 차단할 가능성 → 유료 VPN(Mullvad 등, 월 약 7천원) 또는 주거용 프록시로 전환 준비 필요
2. **사무실 IP 해제**: BUYMA 고객센터에 정식 해제 요청 병행 권장 (장기적으로는 정상 IP 복구가 최선)
3. **요청량 자체 축소**: 현재 하루 3만 건 전체 스캔 구조 → 우선순위 기반으로 필요한 상품만 조회하도록 쿼리 최적화 검토

## 7. 결론

**BUYMA 자동화 시스템은 Cloudflare WARP 도입과 요청 설정 개선을 통해 정상 운영 상태로 복구되었으며, 유사 차단의 재발 방지 조치도 함께 적용되었습니다.**

추가 비용 없이 당일 내 복구 완료. 중장기적으로는 BUYMA 측 IP 해제 요청과 요청량 최적화를 통해 보다 안정적인 운영 체계를 구축할 예정입니다.
