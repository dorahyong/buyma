## 3. 페이지네이션 디버깅 (최장 난관)
BURBERRY KIDS(13개)·MONCLER(163개) 카테고리에서 p2 이후에도 p1 내용 반복 반환.

### 시도한 실패 접근 (전부 무효)
- `?page=N`: 302 리다이렉트 → `?page=1`로 돌려보냄
- `?cp=N`: 200 OK지만 HTML 본문은 여전히 p1
- brand_url의 기존 `?cp=1` 제거 후 재구성
- Referer 헤더에 `?page=N&size=40` 세팅
- URL 로그 + `head=[pnos[:3]]` 디버그 출력
- XHR 인터셉트로 12개 API 호출 캡처 (결과: 아이템 fetch XHR 없음, HTML이 p1 그대로 반환)
- 시크릿 브라우저에서 `?cp=2` 직접 입력 → p1과 동일 확인

### 최종 해결
**DOM 페이지 버튼을 Playwright로 클릭**해야 동작.
- 숫자 버튼 셀렉터: `a[data-shp-contents-type="pgn"][data-shp-contents-id="{N}"]`
- "다음" 버튼: `a[role="button"]:has-text("다음")`
- `wait_for_function`으로 첫 상품 ID 변화 감지 (networkidle 불안정)
- 총 갯수: `"총 N개"` 텍스트 패턴 매칭 (해시 클래스명 회피)
- 3중 종료 조건: max_pages 도달 / 빈 페이지 / 이전 페이지와 완전 중복
