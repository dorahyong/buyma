## 3. 정확한 원인 진단
사용자가 제공한 결정적 단서:
- 매니아온 홈 → 상품 클릭 = ✅ 통과 (캡챠 X, 로그인도 불필요)
- 같은 URL을 주소창에 직접 입력 = ❌ 로그인 강요 → 로그인 후 캡챠

→ **HTTP `Referer` 헤더 체크** 가 원인. playwright의 `page.goto()`는 명시적 referer 인자 없으면 Referer 헤더를 안 보냄 = 주소창 직접 입력과 동일하게 취급됨. smartstore.naver.com이 최근 Referer 체크 강화한 것으로 추정.

- `brand.naver.com` ✅ 영향 없음 (`brand_store_collector` 정상 동작 이유)

## 비개발자용 요약

### 문제
네이버 스마트스토어(매니아온 등) 상품을 자동으로 수집하는 도중, 갑자기 "보안 확인" 화면(캡챠)이 떠서 수집이 멈춤. 신규 추가 10개 + 기존 14개 모두 영향. 브랜드스토어(carpi 등)는 정상.

### 원인
네이버가 최근 스마트스토어 봇 차단 정책을 강화. 일반 사용자가 매장에서 상품을 **클릭해서 들어가면** 정상 노출되지만, 상품 URL을 **주소창에 바로 입력해서 들어가면** 로그인 + 캡챠를 요구함. 자동 수집 프로그램이 후자처럼 동작했기 때문에 봇으로 판단됨.

### 해결
상품 페이지를 열 때 "매장 홈에서 클릭해서 들어왔다"는 정보를 같이 보내도록 프로그램 한 줄 수정. 스마트스토어 14개 캡챠 즉시 해결, 재고/가격 동기화 프로그램도 동일 수정.

### 추가로 발견하여 수정한 문제
네이버 수집기의 "기존 상품 스킵" 옵션이 다른 5개 사이트(카시나, 넥스트젠팩, 라벨루소, 트렌드메카, 옥몰) 수집기와 동작이 달랐음. 이전엔 "한 번이라도 수집한 상품은 무조건 스킵"이었던 것을 "바이마에 등록 완료된 상품만 스킵, 신규+미등록은 매번 갱신"으로 통일. 결과적으로 가격/재고 갱신 누락 위험 제거.

### 잘못된 시도 (참고)
어제 다른 PC에서는 "자동화 흔적 때문에 봇으로 들킴"으로 추정하고 봇 위장 라이브러리 적용 시도했으나 효과 없었음. 일반 크롬 브라우저도 동일하게 캡챠 뜨는 것이 결정적 단서가 되어 정확한 원인 파악 가능.

### 위험 / 한계
지금 적용한 방식은 "매장 홈에서 왔다"고 단순히 알려주는 수준. 네이버가 향후 "정확히 어느 리스트 페이지에서 왔는지 검증" 같은 더 엄격한 체크로 정책 강화하면 다시 막힐 수 있음. 그 시점엔 동일 패턴으로 한 단계 더 정교화 필요.
