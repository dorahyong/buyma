# 작업일지 2026-05-20

## 핵심 성과
카시나(kasina)의 **오프라인 매장 전용(STORE_ONLY) 상품 293건**을 buyma 출품 대상에서 영구 제외. 향후 자동 수집 시점에 차단(collector 사전 필터) + 기존 적재분 정리(buyma 출품 회수 + DB 삭제) 양방향 처리 완료.

---

## 1. 문제 / 배경
- 카시나 일부 상품은 "오프라인 매장 전용(STORE_ONLY)"으로 운영됨 (예: `https://www.kasina.co.kr/product-detail/132656838`)
- 이 상품들은 buyma에서 주문이 들어와도 발송 불가능 → 등록 자체가 위험
- 그러나 기존 수집 파이프라인은 이 플래그를 인지하지 못해 일반 상품과 동일하게 raw → ace → buyma 출품까지 진행됨

## 2. 식별 방법 탐색
처음에는 사용자가 "리스트 페이지의 `<span class='c-flag__item--app'>IN-STORE ONLY</span>`" HTML을 단서로 제시 → 이는 프론트 표시용일 뿐 백엔드 API 응답에서 직접 식별해야 함.

### 검증된 식별 키
- **Shopby list API** (`/products/search`) 응답의 `customProperties` 배열에 다음 객체가 있음:
  ```json
  {"propNo": 1241, "propName": "STORE_ONLY", "propType": "STRING",
   "propValueNo": 14283, "propValue": "STORE_ONLY", "isMultipleSelection": true}
  ```
- detail API 호출 **이전 단계** (list 응답)에서 이미 판별 가능 → 비용 효율적
- 처음 잘못 추정한 `ERP-SHOPBY` 플래그는 위탁(CONSIGNMENT) 표시지 STORE_ONLY와 무관함을 검증

## 3. Collector 수정 (재유입 차단)

| 파일 | 변경 |
|---|---|
| `kasina/kasina_collector.py` 수집 루프 | `model_id` 사전 체크 직후, `fetch_product_detail` 호출 **전**에 `item.customProperties[*].propName == 'STORE_ONLY'` 사전 체크 추가. 해당 시 `SKIP (STORE_ONLY)` 로그 출력 후 detail/options API 호출 자체를 건너뜀 |
| `kasina/kasina_collector.py` `convert_to_raw_data` | 초기 시도였던 detail-기반 STORE_ONLY 체크 블록은 제거 (list 단계에서 이미 판별되므로 불필요) |

영향 범위: `kasina_collector.py`는 단일 진입점이므로 `run_daily_multisource.py`에서 호출되는 카시나 수집도 자동 반영.

## 4. 기존 적재분 식별 (1회성 스크립트)
**`kasina/identify_store_only.py`** — 활성 브랜드 194개에 대해 list API 페이징 순회하며 STORE_ONLY productNo 전수 수집.
- 중간 진행 결과를 `store_only_raw.csv`에 누적 저장 (중간 실패 복구용)
- 첫 실행: NOMANUAL 도달 후 cp949 인코딩 에러로 중단 → `sys.stdout.reconfigure(encoding='utf-8')` 추가
- 두 번째 실행: 매핑 단계에서 SQL IN 절이 너무 길어 MySQL 연결 끊김 → 별도 매핑 스크립트로 분리

**`kasina/map_store_only_to_db.py`** — STORE_ONLY 후보를 우리 DB와 매칭.
- 단일 쿼리로 kasina raw + ace 전수(19,738행) 메모리 로드 후 dict lookup → IN 절 길이 문제 회피

### 식별 결과
- 카시나 전체 STORE_ONLY: **293건**
- 우리 DB와 교집합(정리 대상): **293건 (100%)**
- 브랜드 분포: BAPE 278 / KUTA DISTANCE LAB 12 / ANTI SOCIAL SOCIAL CLUB 2 / ADIDAS 1
- 결과 리포트: `kasina/store_only_report.csv`

## 5. 정리 작업 (BUYMA + DB)
**`kasina/cleanup_store_only.py`** — `buyma_cleaners/cleanup_hazard_products.py` 패턴을 따라 4단계 처리.

| Phase | 내용 | 결과 |
|---|---|---|
| 1 | CSV → DB 재조회로 ace_id/reference_number/is_published 확정 | 검증 293건 |
| 2 | BUYMA Shopper API (`control='delete'`) | **198건 100% 성공, 실패 0** |
| 3 | ace_product_images / options / variants / products 순 삭제 | images 756, options 1,472, variants 1,182, products 293 |
| 4 | raw_scraped_data 삭제 | raw 293 |

안전장치:
- BUYMA 삭제 실패 ace_id는 ace/raw 삭제도 보류 (실제로는 실패 0)
- 사전 확인: kasina raw-ace 관계가 1:N 0건임을 SQL로 확인 → ace 삭제 후 raw 삭제 안전
- `--dry-run` 지원, 실행 전 `DELETE` 키워드 입력 확인

## 6. 검증 (collector 사전 차단 동작)

| 케이스 | 명령 | 결과 |
|---|---|---|
| 전수 STORE_ONLY 차단 | `--brand BAPE --limit 50 --dry-run` | **50/50 SKIP (STORE_ONLY)**. detail/options API 호출 0회 |
| 일반 수집 통과 | `--brand NIKE --limit 5 --dry-run` | **5/5 정상 수집** (model_id/가격/카테고리/재고 정상) |

차단 로직이 detail API 호출 이전에 동작하므로 STORE_ONLY가 많은 BAPE 같은 브랜드는 list API 호출만으로 빠르게 끝남.

## 7. 위험 / 향후 과제
- BAPE는 현재 카시나에서 거의 100% STORE_ONLY 운영(278/278). 향후 BAPE에서 buyma 출품 가능한 상품이 나타나도 list API에서 자동 통과되므로 별도 정책 변경 불필요.
- 동일한 STORE_ONLY 운영 정책이 다른 수집처(okmall/nextzennpack/labellusso)에도 있는지 미확인. 필요 시 동일 패턴으로 확장 가능.
- list API 응답 스키마 의존: 카시나(Shopby)가 향후 `customProperties` 구조나 `propName='STORE_ONLY'` 키를 변경하면 차단이 무력화될 수 있음. 변경 시 collector 한 곳만 수정하면 됨.

## 비개발자용 요약

### 문제
카시나에는 일부 상품이 "**오프라인 매장에서만 살 수 있는** 상품"으로 분류되어 있음 (온라인 주문 시 발송 불가). 그런데 우리 자동 수집 프로그램은 이 표시를 인지하지 못하고 일반 상품과 똑같이 buyma에 출품해 왔음. 손님이 주문해도 보낼 수 없는 상품을 팔고 있었던 셈.

### 식별 방법
카시나의 상품 목록 API 응답을 자세히 분석한 결과, **"STORE_ONLY" 라는 라벨**이 같이 내려온다는 걸 확인. 이 라벨이 붙은 상품은 자동으로 제외하도록 처리.

### 한 일
1. **앞으로의 수집** — 수집 프로그램이 상품 목록을 받을 때 STORE_ONLY 라벨이 있으면 그 자리에서 건너뛰도록 수정.
2. **이미 모인 데이터** — 카시나 전 브랜드를 다시 훑어서 STORE_ONLY 상품 **293건**을 찾아냄.
   - 그 중 **198건이 이미 buyma에 출품된 상태** → buyma에서 모두 회수(삭제) 완료.
   - 나머지 95건은 buyma까지 안 갔지만 우리 DB에 쌓여있던 것 → DB에서 삭제.
3. **검증** — 수정한 수집 프로그램이 STORE_ONLY 상품은 차단하고 일반 상품은 정상 수집하는지 두 브랜드(BAPE / NIKE)로 확인 완료.

### 브랜드별 정리량
- BAPE: 278건 (카시나에서 BAPE 상품은 거의 전량이 매장 전용이었음)
- KUTA DISTANCE LAB: 12건
- ANTI SOCIAL SOCIAL CLUB: 2건
- ADIDAS: 1건

### 위험 / 한계
- 카시나가 향후 STORE_ONLY 표시 방식을 바꾸면 다시 일반 상품과 섞일 수 있음. 그땐 수집 프로그램 한 곳만 다시 손보면 됨.
- 다른 수집처(옥몰, 넥스트젠팩, 라벨루소)에도 비슷한 "오프라인 전용" 분류가 있는지는 미확인 — 필요 시 같은 방식으로 적용 가능.
