  ---                                                                                              목적
                                                                                                 
  orchestrator.py로 buyma 상품 자동 등록 파이프라인을 돌리고 있다. 5단계 파이프라인(COLLECT →    
  CONVERT → IMAGE → PRICE → REGISTER)에서 마지막 REGISTER 단계(buyma_new_product_register.py)가
  buyma API에 상품을 등록할 때, API가 에러를 반환하면서 ace_products.status = 'fail'이 된        
  상품들이 있다.

  이 에러들을 하나씩 고쳐서, 다음에 orchestrator를 돌릴 때 기존 실패 상품들이 성공적으로
  등록되도록 하는 것이 목적이다.

  에러 추출 방법

  ace_products 테이블의 api_response_json 컬럼에 buyma API 응답이 저장되어 있다. status =        
  'fail'인 상품들의 에러 메시지를 분류하면 아래 5가지가 나온다.

  -- 에러별 건수 확인
  SELECT
    CASE
      WHEN api_response_json LIKE '%カテゴリID%' THEN 'category_id 오류'
      WHEN api_response_json LIKE '%options%details%' THEN 'options details keys 오류'
      WHEN api_response_json LIKE '%reference_price%' THEN 'reference_price 오류'
      WHEN api_response_json LIKE '%削除できない%' THEN '削除できない 오류'
      ELSE 'etc'
    END as error_type,
    COUNT(*) as cnt
  FROM ace_products
  WHERE status = 'fail'
  GROUP BY error_type;

-----------------------------------------
  ● Error #3: reference_price < price 정리
  ---
  원인

  buyma API 규칙: reference_price(참고정가) > price(판매가) 필수.
  reference_price = 오케이몰 한국 정가 × 0.1 (÷10 → JPY 환산)
  price = buyma 경쟁자 최저가 - random(1~9)

  한국 정가를 엔화 환산한 값이 buyma 일본 시장 경쟁가보다 낮은 상품에서 발생.
  한국에서 싸게 파는 물건이 일본에서 더 비싸게 팔리는 경우.

  에러 메시지:
  参考価格は販売価格を超える価格を設定してください。

  ---
  가격 흐름

  ┌────────────────────────────┐     ┌──────────────────────────────────────┐
  │ 오케이몰 정가 ₩69,000      │     │ reference_price = 69,000 × 0.1       │
  │ 오케이몰 판매가 ₩49,000    │     │                 = ¥6,900             │
  │ (우리 매입가)               │     │                                      │
  └────────────────────────────┘     │ buyma 경쟁 최저가 = ¥9,902           │
                                     │ price = 9,902 - rand = ¥9,895        │
                                     │                                      │
                                     │ ¥6,900 < ¥9,895 → API 거부          │
                                     └──────────────────────────────────────┘

  핵심: reference_price는 표시용 정가일 뿐, 우리 원가·마진과 무관.
  실제 마진은 price × 9.2 - 수수료 - 매입가 - 배송비로 계산되며,
  403건 중 402건이 마진이 남는 상품.

  ● Error #4: 削除できない商品です 정리
  ---
  원인

  REGISTER(buyma_new_product_register.py)에서 신규 등록 시 전체 품절이면
  control:"delete"를 보내도록 되어 있었음.
  하지만 이 상품들은 buyma에 한 번도 등록된 적 없는 미등록 상품.
  → buyma API가 "削除できない商品です" (삭제할 수 없는 상품) 에러 반환.
  → webhook(server.py)이 이 에러를 받으면 is_active=0 으로 자동 비활성화.
  → 이후 파이프라인에서 영구적으로 제외됨.

  에러 메시지:
  削除できない商品です

  │                                  │ 호출부에서 None 체크 후 API 호출 안함.       │

  ---
  재등록

  940건이 is_active=1로 복원되어 다시 REGISTER 대상에 포함됨.
  전체 품절인 상품은 skip 처리되고, 재고가 있는 상품은 정상 등록됨.
  나중에 SYNC에서 바이마 등록 상품이 전체 품절되면 delete API 전송.
  (SYNC의 delete는 이미 등록된 상품이므로 정상 동작)

  │ 대상     │ buyma 미등록 상품        │ buyma 등록 완료 상품             │
  │ 전체 품절│ skip (API 호출 안함)     │ control:"delete" API 전송       │
  │ 이유     │ 미등록이라 삭제 불가     │ 등록된 상품이므로 삭제 가능      │
  └──────────┴─────────────────────────┴─────────────────────────────────┘
